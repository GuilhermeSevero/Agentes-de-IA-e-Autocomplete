IA, Modelos e LLMs

O que é IA?

  • Inteligência Artificial: Sistemas que simulam capacidades humanas
  • Machine Learning: Aprende com dados
  • Deep Learning: Redes neurais profundas

LLM - Large Language Model:

  • Modelo de Linguagem: Entende e gera texto
  • Treinado em bilhões de textos: Livros, código, web
  • Transformer: Arquitetura neural moderna

Modelo vs LLM:

  • Modelo: Qualquer rede neural treinada
  • LLM: Modelo específico para linguagem (Large Language Model)

Como Funciona um LLM?

┌────────────────────────────────────────────────────┐ │ 1. Input (Prompt do usuário) │ └────────────────────────┬───────────────────────────┘ ↓ ┌────────────────────────────────────────────────────┐ │ 2. Tokenização (quebra em tokens) │ └────────────────────────┬───────────────────────────┘ ↓ ┌────────────────────────────────────────────────────┐ │ 3. Processamento Neural │ │ → Prediz apenas 1 TOKEN │ │ → Modelo processa: [Input + Tokens já gerados] │ └────────────────────────┬───────────────────────────┘ │ │ ↑ ↓ └────┘ Loop de Predição ┌────────────────────────────────────────────────────┐ │ 4. Output (Resposta completa gerada) │ └────────────────────────────────────────────────────┘

🤖 Exemplos de LLMs:

Proprietários: GPT-5, Sonnet, Gemini
Open Source: Llama, Qwen, Mistral, Gemma

Conceitos Práticos

1. Tokens:

  • Tokens: "Palavras" que o modelo entende
  • Texto é quebrado em pedaços menores
  • 32K tokens ≈ 24 mil palavras

2. Temperatura:

  • 0.0: Preciso, determinístico (código, math)
  • 0.7-1.0: Criativo, variado (brainstorm)

3. Fine-tuning:

  • Base model: Conhecimento geral
  • Fine-tuned: Especializado (ex: Qwen-Coder)

4. Treinamento vs Inferência:

Treinamento (não fazemos): • Criar modelo do zero • Supercomputadores • Semanas/meses • $$$ milhões Inferência (fazemos local): • Usar modelo pronto • Seu PC/GPU • Instantâneo • Gratuito

5. Prompt Engineering:

❌ Ruim:

"Me ajuda com código"

✅ Bom:

"Crie função Python para validar email com regex. Inclua docstring e tratamento de erros."

Modelos de IA

Conceitos Essenciais:

  • Parâmetros: Conexões neurais do modelo (7B, 14B, 32B)
  • Quantização: Otimização (Q4, Q8, FP16)
  • Context Window: Quanto o modelo "lembra"

Regra Prática:

VRAM → Modelo Recomendado 8GB → Qwen2.5-Coder 7B Q4 16GB → Qwen2.5-Coder 14B Q4 32GB+ → Qwen3Coder 30B Q4

Matemática dos Modelos

A Fórmula Básica:

  • 1 parâmetro = 4 bytes (float32)
  • 7 bilhões × 4 bytes = 28 GB
  • Mais overheads = ~30 GB total

🧮 Calculadora Rápida:

Tamanho = Parâmetros × Bytes Qwen2.5-Coder 7B × 4 = 28 GB (FP32) Qwen2.5-Coder 14B × 4 = 56 GB (FP32) Qwen3Coder 30B × 4 = 120 GB (FP32) 🤯 GPT-4: 1.8T × 4 = 7.2 TB!

Por que é Problema:

  • GPU consumidor: 8-24 GB VRAM
  • Modelo 7B FP32: 28 GB
  • Não cabe! 😱

A Solução:

  • Quantização: Reduzir precisão
  • Menos bytes = cabe na GPU
  • Pequena perda de qualidade

Quantização: Reduzindo Precisão

Tipos de Quantização:

  • FP32: 32 bits = 4 bytes (original)
  • FP16: 16 bits = 2 bytes (-50%)
  • Q8: 8 bits = 1 byte (-75%)
  • Q4: 4 bits = 0.5 bytes (-87.5%)

Qualidade vs Tamanho:

  • Q8: Quase sem perda
  • Q4: Boa qualidade, muito menor
  • Q2: Existe, mas degrada muito

Qwen2.5-Coder 7B na Prática:

📊 Comparação Real: FP32: 28 GB ← Não cabe FP16: 15 GB ← Cabe em RTX 4090 Q8: 8.1 GB ← Cabe em RTX 3080 Q4: 4.7 GB ← Cabe em GTX 1080 🎯 Sweet spot: Q4 ou Q8

💡 Regra Prática:

Sua VRAM ÷ 2 = Modelo máximo em Q4
Ex: 16 GB → Qwen2.5-Coder 14B Q4

KV Cache: A Memória dos Modelos

"Como os modelos 'lembram' da conversa"

O que é KV Cache:

  • K = Keys: "Chaves" das palavras anteriores
  • V = Values: "Valores" do contexto
  • Cache: Memória temporária rápida
  • Evita recálculo: Não processa tudo novamente

Por que é importante:

  • Conversas longas = muito KV Cache
  • Mais contexto = mais VRAM usada
  • Limite de tokens = limite de memória

Exemplo Prático:

Context Window: 32K tokens Qwen2.5-Coder 7B Q4: ~4.7GB base + 8K tokens contexto = +400MB + 16K tokens contexto = +800MB + 32K tokens contexto = +1.6GB Total: até 5.1GB VRAM!

💡 Dica Prática:

Context window grande = mais VRAM necessária.
Monitore uso de memória em conversas longas!

🎯 ChatGPT vs Open WebUI

Mesma pergunta, velocidade local, privacidade total

Instalando Open WebUI

Opção 1: Standalone

Use quando já tiver Ollama instalado separadamente

# Standalone (conecta ao Ollama externo) docker run -d \ -p 3000:8080 \ --add-host=host.docker.internal:host-gateway \ -v open-webui:/app/backend/data \ --name open-webui \ --restart always \ ghcr.io/open-webui/open-webui:main # Acesse: http://localhost:3000
  • Porta 3000: Interface web
  • host-gateway: Conecta ao Ollama do host
  • Volume: Persiste conversas e configs

Opção 2: Bundle com Ollama

Tudo em um único container (GPU required)

# Bundle: Ollama + Open WebUI docker run -d \ -p 3000:8080 \ --gpus=all \ -v ollama:/root/.ollama \ -v open-webui:/app/backend/data \ --name open-webui \ --restart always \ ghcr.io/open-webui/open-webui:ollama
  • --gpus=all: Acesso à GPU
  • 2 volumes: Modelos + dados
  • Tudo-em-um: Não precisa Ollama separado

Arquitetura Completa

┌─────────────────────────┐ │ 💻 Continue (VSCode) │ │ Autocomplete em tempo │ │ real no seu código │ └──────────┬──────────────┘ │ HTTP REST API │ localhost:11434 ┌──────────▼──────────────┐ │ 🌐 Open WebUI │ │ Interface web chat │ │ http://localhost:8080 │ └──────────┬──────────────┘ ┌──────────▼──────────────┐ │ ⚡ CLIs & Scripts │ │ Claude, Codex, Forge │ │ Automação terminal │ └──────────┬──────────────┘ │ API OpenAI-compatible ┌──────────▼──────────────┐ │ 🚀 Ollama Engine │ │ Port: 11434 │ │ API: /v1/chat │ │ Models: ~/.ollama/ │ └──────────┬──────────────┘ ┌──────────▼──────────────┐ │ 🤖 Modelo LLM │ │ Qwen2.5-Coder (7B) │ │ GPU/CPU Processing │ └─────────────────────────┘

Comunicação:

  • Protocolo: REST API (OpenAI-compatible)
  • Endpoint: http://localhost:11434/v1/chat/completions
  • Modelos: Armazenados em ~/.ollama/models/
  • Processamento: GPU (CUDA/ROCm) ou CPU

Fluxo de Dados:

1. Você escreve código/prompt

↓

2. Cliente envia para Ollama (porta 11434)

↓

3. Ollama carrega modelo na RAM/VRAM

↓

4. Inferência no modelo (GPU/CPU)

↓

5. Resposta volta para o cliente
