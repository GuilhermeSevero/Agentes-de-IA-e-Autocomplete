Modelos de IA

Conceitos Essenciais:

  • Parâmetros: Conexões neurais do modelo (7B, 14B, 32B)
  • Quantização: Otimização (Q4, Q8, FP16)
  • Context Window: Quanto o modelo "lembra"

Regra Prática:

VRAM → Modelo Recomendado 8GB → Qwen2.5-Coder 7B Q4 16GB → Qwen2.5-Coder 14B Q4 32GB+ → Qwen3Coder 30B Q4

Matemática dos Modelos

A Fórmula Básica:

  • 1 parâmetro = 4 bytes (float32)
  • 7 bilhões × 4 bytes = 28 GB
  • Mais overheads = ~30 GB total

🧮 Calculadora Rápida:

Tamanho = Parâmetros × Bytes Qwen2.5-Coder 7B × 4 = 28 GB (FP32) Qwen2.5-Coder 14B × 4 = 56 GB (FP32) Qwen3Coder 30B × 4 = 120 GB (FP32) 🤯 GPT-4: 1.8T × 4 = 7.2 TB!

Por que é Problema:

  • GPU consumidor: 8-24 GB VRAM
  • Modelo 7B FP32: 28 GB
  • Não cabe! 😱

A Solução:

  • Quantização: Reduzir precisão
  • Menos bytes = cabe na GPU
  • Pequena perda de qualidade

Quantização: Reduzindo Precisão

Tipos de Quantização:

  • FP32: 32 bits = 4 bytes (original)
  • FP16: 16 bits = 2 bytes (-50%)
  • Q8: 8 bits = 1 byte (-75%)
  • Q4: 4 bits = 0.5 bytes (-87.5%)

Qualidade vs Tamanho:

  • Q8: Quase sem perda
  • Q4: Boa qualidade, muito menor
  • Q2: Existe, mas degrada muito

Qwen2.5-Coder 7B na Prática:

📊 Comparação Real: FP32: 28 GB ← Não cabe FP16: 15 GB ← Cabe em RTX 4090 Q8: 8.1 GB ← Cabe em RTX 3080 Q4: 4.7 GB ← Cabe em GTX 1080 🎯 Sweet spot: Q4 ou Q8

💡 Regra Prática:

Sua VRAM ÷ 2 = Modelo máximo em Q4
Ex: 16 GB → Qwen2.5-Coder 14B Q4

KV Cache: A Memória dos Modelos

"Como os modelos 'lembram' da conversa"

O que é KV Cache:

  • K = Keys: "Chaves" das palavras anteriores
  • V = Values: "Valores" do contexto
  • Cache: Memória temporária rápida
  • Evita recálculo: Não processa tudo novamente

Por que é importante:

  • Conversas longas = muito KV Cache
  • Mais contexto = mais VRAM usada
  • Limite de tokens = limite de memória

Exemplo Prático:

Context Window: 32K tokens Qwen2.5-Coder 7B Q4: ~4.7GB base + 8K tokens contexto = +400MB + 16K tokens contexto = +800MB + 32K tokens contexto = +1.6GB Total: até 5.1GB VRAM!

💡 Dica Prática:

Context window grande = mais VRAM necessária.
Monitore uso de memória em conversas longas!

Instalando Ollama

Linux (Nativo):

# Instalação rápida curl -fsSL https://ollama.com/install.sh | sh # Download do modelo ollama pull qwen2.5-coder:7b # Testar ollama run qwen2.5-coder:7b

✅ Vantagens Linux Nativo:

Máxima performance • GPU direto • Menos overhead

Docker (Qualquer SO):

# CPU Only: docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama # Nvidia GPU: docker run -d --gpus=all -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama # AMD GPU: docker run -d --device /dev/kfd --device /dev/dri -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama:rocm # Baixar modelo: docker exec -it ollama ollama pull qwen2.5-coder:7b

🐳 Vantagens Docker:

Isolamento • Portabilidade • Fácil backup

Claude Code

Principais Features:

  • Agentic Mode: Executa tarefas complexas automaticamente
  • MCP Servers: Conecta com ferramentas externas
  • Slash Commands: Comandos customizáveis
  • Context-Aware: Entende seu projeto inteiro

# Instalação:

npm install -g @anthropic-ai/claude-code

Uso:

# Modo interativo (padrão) claude # Print direto (pipes/scripts) claude -p "criar API REST em Python" # Continuar última conversa claude -c "adicione validação de email" # Escolher modelo específico claude --model opus "otimizar performance" # MCP servers customizados claude --mcp-config ./mcp-servers.json

Codex CLI

Características:

  • GPT-5: Modelo mais avançado da OpenAI
  • APIs Externas: Também modelos locais

Usando APIs Externas:

  • Conecte com Ollama via API OpenAI-compatible
  • Ou use proxy compatível (OpenRouter)

# Instalação:

npm install -g @openai/codex


# Uso:

codex -c model_provider=openrouter -m z-ai/glm-4.6

Setup (~/.codex/config.toml):

# Usando Ollama (local) [model_providers.ollama] name = "Ollama" base_url = "http://localhost:11434/v1" # Usando Proxy (OpenAI-compatible) [model_providers.openrouter] name = "OpenRouter" base_url = "https://openrouter.ai/api/v1" env_key = "OPENROUTER_API_KEY"

Gemini CLI

Vantagens:

  • Free Tier: 60 requests/min e 1.000 requests/dia
  • Custom Commands: Crie seus próprios comandos
  • Multi-modal: Texto, imagem, código
  • Context Window: Gemini 2.5 Pro com context window de 1M token

# Instalação:

npm install -g @google/gemini-cli


# Uso:

gemini

Como Criar Comandos:

(~/.gemini/commands/git/commit.toml)

# In: description = "Generates a Git commit message based on staged changes." # The prompt uses !{...} to execute the command and inject its output. prompt = """ Please generate a Conventional Commit message based on the following git diff: ```diff !{git diff --staged} ``` """

QwenCode CLI

"Fork do Gemini CLI otimizado para Qwen-Coder models"

Vantagens:

  • Free Tier: 60 requests/min e 2.000 requests/dia
  • Custom Commands: Crie seus próprios comandos
  • APIs Externas: Também modelos locais

# Instalação:

npm install -g @qwen-code/qwen-code@latest


# Uso:

qwen

Setup:

# Environment variables export OPENAI_API_KEY="your_api_key_here" export OPENAI_BASE_URL="your_api_endpoint" export OPENAI_MODEL="your_model_choice"

Diferenças do Gemini CLI:

  • 100% Local: Foca em modelos Qwen3 Open Source
  • Code-First: Otimizado para código
  • Features: Maioria das features do Gemini estão presentes
  • Setup: Configurações parte da pasta "~/.qwen"
